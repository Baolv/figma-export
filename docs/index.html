⚙️ How it works

The Figma Plugin API runs inside Figma with full access to all design data — variables, styles, component properties, and asset exports. When you click Export & Download, the plugin bundles everything into a ZIP file and downloads it directly to your machine. No server, no terminal, no extra software needed.

💡

The ZIP contains a folder named after your Figma file key. Extract it and move the folder to ~/figma-exports/ — that's the one manual step between clicking Export and using the data with your AI agent.

🚀 Setup guide

ℹ️

You need the Figma desktop app to install local plugins. The browser version doesn't support "import from manifest". Download it free at figma.com/downloads.

1

Import into Figma desktop

  1. Open the Figma desktop app
  2. Open any design file (plugins only appear inside files, not the home screen)
  3. Press ⌘ / to open Quick Actions
  4. Type "Import plugin from manifest" and press Enter
  5. Select this file: ~/Projects/figma-export/figma-plugin/manifest.json
✅

Once imported, the plugin is available in all your Figma files. You only need to import it once. No build step required — the plugin is pre-built.

2

Create the exports folder

This is where you'll move extracted exports. The default path is ~/figma-exports — create it once:

Terminal
mkdir -p ~/figma-exports
📁

Want a different folder? (e.g. a shared Dropbox or project folder) — run this instead, then skip the mkdir above:

Terminal — optional, one-time
# 1. Create your preferred folder
mkdir -p ~/Dropbox/figma-exports

# 2. Tell your AI agent to look there
echo '{"exportsDir":"~/Dropbox/figma-exports"}' > ~/.figma-export.json

Your AI agent reads ~/.figma-export.json automatically — no changes to CLAUDE.md needed. This file lives in your home directory and is never committed to git.

3

Install the figma-lookup skill

Copy the skill into your project's .claude/skills/ folder. Use it with /figma-lookup <url> or /figma-lookup <url> implement. The skill handles URL parsing, node lookup, staleness checking, device chrome detection, and smart asset matching.

Terminal
mkdir -p .claude/skills
cp -r ~/Projects/figma-export/skills/figma-lookup .claude/skills/
ℹ️

The SKILL.md format is supported by all major coding agents. To install globally instead of per-project, copy to ~/.claude/skills/. If your agent doesn't support skills, paste the contents of skills/figma-lookup/SKILL.md into its rules file (e.g. .cursorrules).

🖼️ Assets — how the plugin detects icons vs images

The plugin needs to decide for every node in your design: is this an icon/vector, a raster image, or just layout (a frame with children to walk)? It uses four signals in priority order.

Export settings only Default

Exports only layers you marked in Figma's Export panel. Format and scale come from Figma — no extra config in the plugin.

Use when: Your designers mark their assets in Figma. Zero guessing, zero false positives.
Auto detect

Also exports by fill type, node type, and layer name — even if no export settings are set.

Use when: The file has no export settings and you want the plugin to figure it out automatically.
⭐

Recommended workflow: Use Export settings only and mark your assets in Figma's Export panel (right panel → Export → +). This gives you full control — format, scale, suffix — and the plugin respects it exactly. No naming conventions needed.

1
Designer set export settings in Figma Most reliable

In Figma's right panel → Export section → click + to add a format. The plugin reads this and exports using exactly those settings — format, scale, suffix — overriding everything else.

What to do in Figma
1. Select any layer (icon, image, component — anything)
2. Right panel → scroll to "Export" section
3. Click + → choose format (SVG / PNG / JPEG / PDF) and scale
4. Done — plugin will always export this node correctly

Works for: ANY node type, ANY name. This is how the eye-slash icon, the chevron dropdown, and the checkmark circle should all be marked.

2
Image fill fill.type === "IMAGE"

When a designer places a photo into a shape (File → Place Image, or drag-and-drop), Figma stores it as an IMAGE fill. The plugin detects this automatically — no naming or export settings needed.

Catches: hero images, product photos, screenshot backgrounds, illustration fills. Exported using the Images format settings in the plugin UI.

3
Primitive vector node type

Some Figma node types are always vectors — no ambiguity, no settings needed:

VECTORBOOLEAN_OPERATIONSTARPOLYGON

The Vector node (eye-slash icon in your password field) is detected this way automatically. Exported using the Icons & Vectors format settings.

4
Name heuristic — last resort Brittle

If none of the above match, the plugin checks the layer name for /icon|logo|glyph/i on INSTANCE or GROUP nodes.

Matches
✓ "sign up/login icons"   ← contains "icon"
✓ "icon-search"
✓ "logo-google"
Misses
✗ "chevron"               ← generic name, missed
✗ "Frame 8889779"         ← no match
✗ "eye-slash"             ← not in the pattern

Fix: use Priority 1 — add export settings in Figma's Export panel on these layers.

What about FRAME nodes?

Frames are never treated as asset leaves — even if they have export settings or contain only icons. A FRAME is layout: it has children (text, icons, other frames) that the AI needs to understand as structured data, not a flat graphic. The plugin always walks into frames and detects assets inside them.

⚠️

In your screenshot: Frame 8889779 is the container around the eye-slash icon. The plugin walks into it, finds the Vector child, and exports that as an SVG — not the whole frame.

Export formats by asset type

Icons & Vectors

Available: SVG (recommended), PNG (with scales), PDF.
JPEG excluded — lossy compression degrades sharp vector edges.

Default: SVG only. Add PNG @2x/@3x if your project can't use inline SVG.
Images

Available: PNG, JPEG (with scales), PDF.
SVG excluded — raster photos can't be converted to SVG.

Default: PNG @2x + @3x. Use JPEG for photos where file size matters.

PNG and JPEG scales

@1x

1 pixel per point. Smallest file, standard resolution.

Use for: Prototypes, web non-retina, or when file size matters.
@2x Default

2 pixels per point. Retina / high-DPI. Standard for iOS and most web.

Use for: Most apps — covers standard and retina screens.
@3x Default

3 pixels per point. Super-retina. iPhone 12 Pro+, Pixel phones.

Use for: iOS apps targeting newer high-res devices.

▶️ Running an export

1

Open your design file in Figma desktop

Navigate to the page that contains the screens you want to export.

2

Launch the plugin

Press ⌘ / in Figma → type "Design Export for AI Agents" → press Enter.

Alternatively: right-click the canvas → Plugins → Development → Design Export for AI Agents.

3

Choose your options and click Export & Download

  • Pick your Scope (usually "Everything on this page")
  • Choose Export settings only (default) or Auto detect for assets
  • Click "Export & Download"

A progress bar shows each step. When finished the browser downloads a .zip file named after your Figma file.

4

Extract and move the folder

Double-click the downloaded .zip to extract it. You'll get a folder named with the file key (e.g. ABC123/ for a cloud file, or a slug like zalora-app/ for a local draft). Drag that folder into ~/figma-exports/.

🔁

Re-exporting the same file? Extract the new ZIP and drag the folder to ~/figma-exports/ — macOS will ask "Replace?". Click Replace. The old export is gone, the new one is in.

📂

Multiple Figma files? Each file gets a different folder name, so they coexist in ~/figma-exports/ without conflict.

🤖 Using with your AI agent

Once exported, your AI agent can implement any screen from just its Figma URL. No extra steps needed — the agent rule handles the rest automatically.

Basic usage

AI agent prompt
Implement this screen as a SwiftUI view:
https://www.figma.com/design/ABC123/MyApp?node-id=456-789

The agent will:

  1. Parse ABC123 (file key) and 456-789 (node id) from the URL
  2. Read ~/figma-exports/ABC123/views/456-789.json for the screen's tree
  3. Read ~/figma-exports/ABC123/tokens.json for colors, spacing, and typography
  4. Look up referenced components in components.json
  5. Use real icons from assets/*.svg and images from assets/*.png
  6. Implement the UI in your project's framework, reusing existing components where they exist
💡

The AI maps the framework-agnostic JSON to whatever framework your project uses — SwiftUI, React, Flutter, etc. It reads the project context automatically.

Getting the node URL from Figma

To get the URL for a specific frame:

  1. Click the frame in Figma
  2. Right-click → Copy link to selection
  3. Paste directly into your AI agent — the URL will have ?node-id=456-789 at the end

If the files are missing

If your AI agent can't find the export files, it means you haven't run the plugin for that file yet — or you ran it but forgot to move the extracted folder to ~/figma-exports/. Re-export, extract the ZIP, and move the folder.

📁 Output structure

Exports land under ~/figma-exports/<file_key>/, normalized so components and tokens are defined once and views reference them by id.

~/figma-exports/
  code-maps/
    ABC123.json     # NEW: Figma component → project file (created by skill; outside the folder so re-exports can't delete it)
  ABC123/             # replaced wholesale on re-export
    meta.json            # file name, pages, view index — now includes exportedAt timestamp
    tokens.json         # resolved design tokens per mode (light/dark)
    styles.json         # color, text, and effect styles
    components.json    # all components, defined ONCE
    node-index.json  # NEW: any node-id → which view file contains it
    views/
      456-789.json     # "Log in" screen
      456-789.png      # NEW: screenshot of the view (capped at 800px wide)
      456-812.json     # "Sign up" screen
      456-812.png      # NEW
    components/       # NEW folder
      16102-7164.png  # NEW: full-res PNG per component (icons, buttons)
    assets/
      icon-google.svg  # icon exported as SVG
      hero-bg@2x.png   # image at 2x resolution
      hero-bg@3x.png   # image at 3x resolution

🔬 Data format

All files are plain JSON. Here's what a typical view entry looks like — a frame from your "Log in" screen:

views/456-789.json (excerpt)
{
  "name": "login form",
  "type": "FRAME",
  "layout": {
    "mode": "VERTICAL",
    "itemSpacing": 16,         // exact gap between children
    "padding": { "t": 24, "r": 20, "b": 24, "l": 20 }
  },
  "children": [
    {
      "name": "sign up/login icons",
      "type": "INSTANCE",
      "component": "Icons/Social",   // defined once in components.json
      "asset": "assets/icon-google.svg" // real file
    },
    {
      "name": "Label text",
      "type": "TEXT",
      "text": "Mobile Number",
      "font": {
        "token": "text.body",         // semantic style name
        "family": "Inter",
        "size": 14, "weight": 500, "lineHeight": 20
      },
      "color": { "token": "color.text.secondary", "hex": "#6b7280" }
    }
  ]
}
tokens.json (excerpt)
{
  "colors": {
    "color.primary": { "light": "#FF6B35", "dark": "#FF8F5E" },
    "color.text.secondary": { "light": "#6b7280" }
  },
  "numbers": {
    "spacing.md": { "light": 16 },
    "radius.card": { "light": 12 }
  },
  "modes": ["light", "dark"]
}

❓ FAQ

Do I need a paid Figma plan? ▾

No. The plugin uses the Figma Plugin API which is available on all plans, including the free plan. It does not use Dev Mode or the REST API.

Do I need to re-export when the design changes? ▾

Yes — re-run the plugin after design changes. Each export overwrites the previous one for that file key, so you always get fresh data. It only takes a few seconds for a single page.

What if my components are in a separate Figma library file? ▾

The plugin walks component instances transitively and exports the anatomy of each referenced component. Even if the master component lives in another file, the plugin resolves it via Figma's API and includes it in components.json. You don't need to export the library file separately.

The plugin exported but my AI agent can't find the files — why? ▾

Two likely causes: (1) The file key in the URL doesn't match the export folder — check your exports directory to see what folders exist. (2) You're using a custom path via ~/.figma-export.json but forgot to move the extracted folder there. Your AI agent reads ~/.figma-export.json for the base path — make sure it matches where you actually put the folder.

What does "Mode 1" mean in tokens.json? ▾

In Figma, every variable collection has at least one mode — this is how Figma handles design token variants like light/dark theme or brand variations. "Mode 1" is Figma's default name when the collection was never renamed. It means your file has a single set of token values with no theming set up yet.

If the team renames the mode in Figma (right-click the collection → rename mode → "light") and adds a "dark" mode, the next export will show "light" and "dark" as mode keys instead of "Mode 1". The plugin exports whatever modes exist in the file automatically.

I work on multiple Figma files — do they overwrite each other? ▾

No. Each file gets its own folder named by its unique key (e.g. ~/figma-exports/ABC123/ and ~/figma-exports/DEF456/). They coexist in ~/figma-exports/ without conflict. Re-exporting the same file replaces only that file's folder when you drag and choose "Replace".

My file is a local draft — the folder is named after the file title, not a key. ▾

Local/unsaved Figma files don't have a real file key, so the plugin uses a slug of the file name instead (e.g. "Zalora App" → zalora-app/). Since local files also have no shareable URL, you'd tell the AI the design name directly — "use my figma export for the Zalora App design" — and it will find the matching folder by scanning meta.json files. Save the file to Figma cloud to get a real key and URL.

Which UI frameworks does this work with? ▾

All of them. The export is framework-agnostic JSON. Any AI agent reads it and maps to whatever framework your project uses — SwiftUI, React, Flutter, Vue, etc. — reusing your existing components and tokens where they already exist in the codebase.

Why is the wrong image showing as the hero? ▾

The node may have an image fill that couldn't be exported separately. Reference the view PNG for visual context. The actual image content needs to come from the app's asset pipeline (CMS, CDN, etc.) — the exported PNG shows what the design intends, but the real asset should be sourced from your content system.

The status bar from the Figma design is showing in my implementation. ▾

That's design chrome, not app UI — the OS renders the real one. The figma-lookup skill auto-detects and skips these nodes ("status bar", "Bars/Status/*", "home indicator") and notes the platform API to configure instead.
