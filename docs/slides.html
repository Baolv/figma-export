What it is

A Figma plugin that exports the full structured design — tokens, layout, spacing, components, real icons/images, and per-screen screenshots — as a ZIP. A Claude skill reads it and implements the screen in your codebase.

Exact spacing & tokens
Real SVG / PNG / PDF assets
Per-screen JSON + screenshot
Any framework: SwiftUI · Android · React · Flutter

Setup — once, ~2 minutes

1
Import the pluginFigma desktop → ⌘/ → "Import plugin from manifest" → pick figma-plugin/manifest.json
2
Make the exports foldermkdir -p ~/figma-exports
3
Install the skillcp -r skills/figma-lookup .claude/skills/ (or into ~/.claude/skills/ globally)

Daily workflow

  • 1
    Export the page once from Figma (the plugin), drag the folder to ~/figma-exports/.
  • 2
    Copy a frame's link in Figma → Copy link to selection.
  • 3
    In Claude Code: /figma-lookup <url> implement with swiftui

What the agent actually reads

  • · Exact layout, padding, item spacing
  • · Design tokens (color / type / spacing)
  • · Components + variants, defined once
  • · Real icons/images; matched to existing project assets
  • · Per-screen screenshot for visual context
  • · Node index → any link resolves, even nested

Sizes always come from the JSON — the screenshot is visual reference only.

Good to know

  • •
    Re-export when the design changes.The skill warns if an export is > 14 days old.
  • •
    Folder is named by the Figma file name (e.g. nc-journey/), not the URL key.Figma only gives the real key to private plugins — the skill matches by name automatically.
  • •
    The code-map grows as you go.Once a component maps to a project file, every future screen reuses it.
